We replicate the core results of Aruoba & Drechsel (2024) and extend them with three LLM-based contributions: (1) LLM identification of 595 economic concepts (vs. 296 original), (2) direct extraction of risk asymmetry signals from Greenbook narrative text, and (3) penalized regression comparison (Ridge vs. LASSO vs. Elastic Net) on both dictionary and LLM feature sets. Additionally, we design a robustness check framework for testing the ±10-word window assumption.

2. The Original Paper & Our Extensions

AD(2024) Pipeline

  1. Select 296 economic concepts via n-gram frequency + manual curation
  2. Score sentiment for each concept using Loughran-McDonald dictionary (±10-word windows)
  3. Compute PC1 of the 296 sentiment indicators
  4. Ridge-regress FFR target changes on Greenbook numerical forecasts + sentiment PC1
  5. Use residuals as monetary policy shocks (R² = 0.94 for full specification)

Key Lessons Learned

  • LLM free-naming + fuzzy matching fails: only ~5.7 non-zero scores per meeting out of 595 concepts. Fixed-concept-list prompting is needed.
  • First-stage vs. second-stage confusion: the paper's Table 1 (FFR changes, R²≈0.94) and Table 2 (forecast errors, R²≈0.25) test different things. Extensions should target Table 1.
  • Full-text API calls are slow: 50K+ tokens of Greenbook text per meeting is impractically slow. Greenbook1-only is a reasonable design choice for full-context LLM scoring.

LLM Sentiment Scoring: Methodology & Limitation

We used a free-naming + fuzzy-matching approach: the LLM read each Greenbook document and freely identified concepts with sentiment scores, which were then matched back to the 595-concept master list at 85% similarity threshold. This produced sparse results (avg. 5.7 non-zero scores per meeting out of 595), because the LLM typically named only 20–40 concepts while the master list contained 595. The sparse matrix failed to produce a meaningful PC1 for the first-stage regression.

Lesson: A fixed-concept-list approach (giving the LLM the exact concept list and requiring scores for every concept) is needed for dense, comparable sentiment matrices. We designed this approach but did not execute it due to API time constraints (≈1.5 hours for 216 meetings).

Key Findings

  • Ridge dominates: Best or near-best across all specifications. Validates AD(2024)'s finding — features exhibit collinearity that Ridge handles well.
  • LLM competitive with dictionary: Ridge R² 0.659 (LLM) vs 0.699 (Dict). Despite sparse matrix issues, LLM features are competitive.

  • Adding forecasts helps consistently: +~0.02 R² across all methods.

7. Robustness Check Design: Testing the ±10-Word Window

The dictionary method scores sentiment within ±10 words of each concept mention, implicitly assuming 20 words of local context are sufficient. We designed a framework to test this assumption by having an LLM read the entire Greenbook1 document (8K–24K words of prose) and score the same 296 concepts with full document context.

Design rationale: This is NOT "LLM replacing dictionary." It is a test of whether the ±10-word window is a binding constraint on information extraction. Greenbook1 (the staff qualitative assessment) provides orders of magnitude more context than 20 words. If full-context LLM scoring cannot improve on window-based dictionary scoring, the narrow window is not the bottleneck. The full design specification and implementation-ready scripts are at src/01a_llm_sentiment_v2.py and results/robustness_checks.ipynb.

Not executed: The full-context LLM scoring requires ≈216 API calls at ≈20–30 seconds each (≈1–1.5 hours total). This was deferred due to time constraints, but the implementation is complete and ready to run when API budget permits.
